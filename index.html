HTML

  • The head element contains important information for the browser, but this information is not diplayed visually to the user.
  • The body element is where the visible information for the end user is contained.
  • If an elements name describes its content and the role on the webpage, it is considered a semantic element.
  • The anchor element is how hyperlinks are defined in html.

CSS

  • Inline CSS, internal CSS, and external CSS, are all ways to style using CSS.
    • Inline: a way to style an HTML element directly in the HTML file.
    • Internal: embed a style sheet directly into HTML file using the style element.
    • External: dedicate a file containing only CSS rules. This is the most common and preferred way.
  • The CSS box model is how CSS styles HTML content. Think of it like a picture frame; margin is the space outisde of the border, the border is the frame, the padding is the space between the border and content, and the content can be thought of as the picture itself.
  • Margin, border, and padding are all CSS properties that can be assigned values.

GIT

  • git status: checks what branch you are currently on.
  • git checkout -b branch-name: creates new branch and switches to it.
  • git pull origin main: insures main branch is up to date on local machine with the remote github repo.
  • git push origin branch-name: pushes local changes to github repo, including the working branch you've created.
  • git add -A: adds changes to staging area.
  • git commit -m "message here": staged changes are commited with message for clarity.
